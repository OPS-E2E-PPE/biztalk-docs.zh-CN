# <a name="phases-of-a-performance-assessment"></a>性能评估阶段
主要目标之一[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]是尽可能容纳尽可能多的处理方案提供最大的灵活性。 由于此极大的灵活性，面对的 BizTalk 解决方案的开发人员的主要挑战之一是确定如何在中提供的功能的最佳使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]为了满足其业务需求。 优化 BizTalk Server 解决方案的性能时，这种灵活性也带来一个挑战。  
  
 本部分介绍的方法，用于通过参与 BizTalk Server 性能评估优化 BizTalk Server 解决方案的性能。 BizTalk Server 性能评估用于最大化特定的 BizTalk Server 解决方案的性能。 为了获得 BizTalk Server 性能评估的最大好处，性能评估应该被划分为以下五个不同步骤/的阶段：  
  
1.  范围  
  
2.  计划  
  
3.  准备  
  
4.  生成实验室  
  
5.  Execute  
  
 本主题介绍每个更详细介绍这些阶段。  
  
 ![性能评估流程阶段](../technical-guides/media/assessmentprocess.gif "AssessmentProcess")  
BizTalk Server 性能评估的阶段  
  
## <a name="in-this-section"></a>本节内容  
  
-   [阶段 1： 范围评估](../technical-guides/phase-1-scoping-the-assessment.md)-介绍建立性能评估的作用域时应遵循的步骤。  
  
-   [阶段 2： 规划评估](../technical-guides/phase-2-planning-the-assessment.md)– 介绍如何创建解决方案里程碑时间线。 使用这种方式主计划以清楚地说明时应满足进行测试解决方案的预期的目标。  
  
-   [阶段 3： 准备评估](../technical-guides/phase-3-preparing-for-the-assessment.md)– 介绍如何提供详细的体系结构关系图的解决方案和解决方案所使用的硬件配置的特定方面。  
  
-   [阶段 4： 构建评估环境](../technical-guides/phase-4-building-the-assessment-environment.md)– 描述的硬件和根据详细的设计与以前已建立的解决方案平台的应用程序的安装。  
  
-   [阶段 5： 执行评估](../technical-guides/phase-5-executing-the-assessment.md)– 提供有关生成通过自动的负载测试和如何检测并消除解决方案中的任何瓶颈的性能数据的信息。
