# <a name="using-scripts-to-deploy-applications"></a>使用脚本来部署应用程序
应使用脚本来部署 BizTalk 应用程序，在可能的情况。 脚本在部署过程中减少人为错误的风险。 你应记录您的部署过程中深度。 你应记录任何内容你不编写脚本非常详细的步骤。 这包括记录到外部系统和非 Microsoft 组件部署到的任何更改。  
  
## <a name="using-btstask"></a>使用 BTSTask  
 可以使用 BTSTask.exe 编写脚本的 BizTalk 应用程序，以及导入现有创建[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)].msi 程序包。 如果创建的应用程序编写的脚本，然后包可以自动生成的生成服务器上使用自动化的过程。
