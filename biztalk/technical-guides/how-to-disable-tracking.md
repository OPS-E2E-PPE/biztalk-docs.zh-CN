# <a name="how-to-disable-tracking"></a>如何禁用跟踪
本主题介绍如何禁用跟踪使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]管理控制台。 可以在业务流程的运行时配置各种跟踪选项、 发送端口、 接收端口和管道使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]管理控制台。 您随时可以在不中断业务流程的情况下更改某项的跟踪选项。  

## <a name="prerequisites"></a>必要條件  
 若要执行本主题中的过程，必须是 BizTalk Server Administrators 组的成员的帐户登录。 详细了解权限的详细信息，请参阅[用于管理应用程序的权限](../technical-guides/permissions-for-managing-an-application.md)。  

> [!NOTE]
>  如果只想要查看的跟踪选项，你可以以登录的成员[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]Operators 组。  

##  <a name="BKMK_DisableOrchTracking"></a> 若要禁用对业务流程的跟踪  

1. 单击**启动**，单击**所有程序**，单击**Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然后单击**BizTalk Server 管理**。  

2. 在控制台树中，展开[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]**管理**，展开 BizTalk 组，展开**应用程序**，然后展开包含所需的业务流程的应用程序配置跟踪。  

3. 单击**业务流程**，然后在右窗格中，右键单击你想要配置跟踪，然后单击该业务的流程**属性**。  

4. 单击**跟踪**选项卡上，按下表中所述方式禁用跟踪选项，然后单击**确定**。  

   |使用此选项|执行的操作|  
   |--------------|----------------|  
   |**跟踪事件-业务流程开始和结束**|清除此复选框可禁用业务流程实例的跟踪之前和之后的整个业务流程处理。|  
   |**跟踪事件-消息发送和接收**|清除此复选框可禁用跟踪消息的发送和接收事件。 此复选框才可用**业务流程开始和结束**复选框。|  
   |**跟踪事件-形状开始和结束**|不需要调试业务流程调试器中的业务流程实例时，请清除此复选框。 选中此复选框之后，将会填充业务流程调试器中的事件列表。 此复选框才可用**业务流程开始和结束**复选框。|  
   |**跟踪消息正文-业务流程处理前**|清除此复选框可以禁用保存和跟踪业务流程实例在处理之前的实际消息内容。 此复选框才可用**消息发送和接收**复选框。|  
   |**跟踪消息正文-业务流程处理之后**|清除此复选框可以禁用保存和跟踪在处理后的实际消息内容的业务流程实例。 此复选框才可用**消息发送和接收**复选框。|  
   |**跟踪消息属性-传入的消息**|清除此复选框可禁用入站消息的升级属性的跟踪。 此复选框才可用**消息发送和接收**复选框。|  
   |**跟踪消息属性-传出消息**|清除此复选框可禁用出站消息的升级属性的跟踪。  此复选框才可用**消息发送和接收**复选框。|  

### <a name="to-disable-tracking-for-a-send-port"></a>若要禁用跟踪的发送端口  

1. 单击**启动**，单击**所有程序**，单击**Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然后单击**BizTalk Server 管理**。  

2. 在控制台树中，展开[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]**管理**，展开 BizTalk 组，展开**应用程序**，然后展开包含你想要配置的发送端口的应用程序跟踪。  

3. 单击**发送端口**，右键单击发送端口，单击**属性**，然后单击**跟踪**。  

4. 下表中所述禁用跟踪选项，然后单击**确定**。  

   |使用此选项|执行的操作|  
   |--------------|----------------|  
   |**跟踪消息正文-端口处理前请求消息**|清除此复选框可以禁用保存和跟踪的消息内容之前接收消息。 **注意：** 必须启用消息正文管道跟踪成功跟踪端口处理前的响应消息。|  
   |**跟踪消息正文-端口处理后请求消息**|清除此复选框可以禁用保存和跟踪的消息内容后接收消息。|  
   |**跟踪消息正文-端口处理前响应消息**|清除此复选框可以禁用保存和跟踪的消息内容之前将消息发送。 此复选框仅适用于要求响应发送端口。|  
   |**跟踪消息正文-端口处理后响应消息**|清除此复选框可以禁用保存和跟踪的消息内容后将消息发送。 此复选框仅适用于要求响应发送端口。|  
   |**跟踪消息属性-端口处理前请求消息**|清除此复选框可禁用入站消息的升级属性的跟踪。|  
   |**跟踪消息属性-端口处理后请求消息**|如果不希望跟踪出站消息的升级的属性，请清除此复选框。|  
   |**跟踪消息属性-端口处理前响应消息**|清除此复选框可以禁用保存和跟踪消息属性，发送消息之前。 此复选框仅适用于要求响应发送端口。|  
   |**跟踪消息属性-端口处理后响应消息**|清除此复选框可以禁用保存和发送消息之后跟踪的属性。 此复选框仅适用于要求响应发送端口。|

4. 下表中所述禁用跟踪选项，然后单击**确定**。  

   |使用此选项|执行的操作|

### <a name="to-disable-tracking-for-a-policy"></a>若要禁用跟踪的策略  

1. 单击**启动**，单击**程序**，单击**Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然后单击**BizTalk Server 管理**。  

2. 在控制台树中，展开[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]**管理**，展开 BizTalk 组，展开**应用程序**，然后展开包含你想要配置的策略的应用程序跟踪。  

3. 单击**策略**，右键单击该策略，单击**属性**，然后单击**跟踪**。  

4. 下表中所述禁用跟踪选项，然后单击**确定**。  

   |使用此选项|执行的操作|  
   |--------------|----------------|  
   |**快速活动**|清除此复选框可禁用运行该策略的实例数据跟踪。|  
   |**条件评估**|清除此复选框可禁用跟踪所选策略中条件的真/假结果。|  
   |**规则触发**|清除此复选框可禁用跟踪操作的该策略最终触发。|  
   |**议程更新**|清除此复选框可禁用跟踪议程的更新。 议程包含一系列值为“True”并需要触发的操作。|  

### <a name="to-disable-tracking-for-a-schema"></a>若要禁用对架构的跟踪  

1. 单击**启动**，单击**程序**，单击**Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然后单击**BizTalk Server 管理**。  

2. 在控制台树中，展开[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]**管理**，展开 BizTalk 组，展开**应用程序**，然后展开包含你想要配置的架构的应用程序跟踪。  

3. 单击**架构**，右键单击该架构，然后单击**属性**。  

4. 在左窗格中，单击**跟踪**。  

5. 执行下述操作以指定要用于跟踪消息，禁用的属性之一，然后单击**确定**:  

   -   清除**始终跟踪所有属性**复选框，如果您不想要使用任何架构版本的所有消息属性。 此复选框仅对文档架构可用。  

   -   清除**选择所有消息属性**复选框，如果您不想要使用所有列出的属性。  

   -   下**属性列表**，清除不想要使用的每个属性的复选框。  

   > [!NOTE]  
   >  应选择所需的选项，因为跟踪消息创建性能和存储有关您的系统开销。  

### <a name="to-disable-tracking-for-a-pipeline"></a>若要禁用跟踪的管道  

1. 单击**启动**，单击**程序**，单击**Microsoft** [!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]，然后单击**BizTalk Server 管理**。  

   > [!NOTE]  
   >  如果设置管道的跟踪选项，则还将设置为使用的管道的每个端口全局跟踪选项。 此操作，反过来，可能会导致的数据远远多于预期，跟踪这将降低系统性能。 相反，您可以设置跟踪选项上的发送端口和接收端口。  

2. 在控制台树中，展开[!INCLUDE[btsBizTalkServer2006r3ui](../includes/btsbiztalkserver2006r3ui-md.md)]**管理**，展开 BizTalk 组，展开**应用程序**，然后展开包含你想要配置的管道的应用程序跟踪。  

3. 执行以下操作之一：  

   -   若要禁用跟踪的默认 BizTalk 管道，请展开\<的所有项目\>。  

   -   若要禁用跟踪已部署到 BizTalk 应用程序的自定义管道，请展开包含管道的应用程序。  

4. 单击**管道**文件夹，右键单击该管道，然后单击**跟踪**。  

   > [!NOTE]  
   >  若要禁用跟踪在一次，多个管道按住 Shift 键，单击要配置每个管道，右键单击一个管道，然后依次**跟踪**。  

5. 下表中所述禁用跟踪选项，然后单击**确定**。  


   |                使用此选项                |                                                                                                                                                                                                                                                                                                                                 执行的操作                                                                                                                                                                                                                                                                                                                                 |
   |----------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   |     **端口开始和结束事件**      |                                                                                                                                                                                                                                                                  清除此复选框可禁用跟踪仅一个实例开始和结束时。 详细信息包括项名称、程序集和其他元数据。                                                                                                                                                                                                                                                                  |
   |  **消息发送和接收事件**   |                                                                                                                                                                                                                                                      清除此复选框可禁用跟踪的消息发送和接收事件。 此复选框才可用才**端口开始和结束事件**处于选中状态。                                                                                                                                                                                                                                                       |
   | **管道处理前的消息** | 清除此复选框可以禁用保存和跟踪的管道收到的消息正文，其中包含元数据，例如 Url 和已升级属性。 如果是接收管道，则消息正文部分是传输组件提交给该管道的原始消息。 根据相应的应用程序，可能对该消息进行加密、签名或编码。 当使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]映射中，如果这是接收管道中，跟踪后就会发生处理入站的映射。<br /><br /> 此复选框才可用才**消息发送和接收事件**处于选中状态。 |
   | **管道处理后的消息**  |                        清除此复选框可以禁用保存和跟踪管道发送的消息正文，其中包含元数据，例如 Url 和已升级属性。 如果是接收管道，则消息正文部分是提交给 MessageBox 数据库的已处理消息（它可以是 XML，这取决于相应的应用程序）。 当使用[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]映射中，如果这是发送管道之前, 进行跟踪处理出站映射。<br /><br /> 此复选框才可用才**消息发送和接收事件**处于选中状态。                        |
