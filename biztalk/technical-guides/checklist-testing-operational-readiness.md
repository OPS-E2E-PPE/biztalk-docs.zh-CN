# <a name="checklist-testing-operational-readiness"></a>清单： 测试操作的准备情况
对操作的准备情况测试是非常小执行或经常被忽视的重要过程。 彻底的测试是一项关键要求的任何企业级别的应用程序，并使用一个解决方案开发[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]也不例外。 至少，你应执行以下测试的将 BizTalk 解决方案移到生产环境之前：  
  
|步骤|参考|  
|-----------|---------------|  
|单元测试|[执行单元测试](../technical-guides/performing-unit-testing.md)|  
|功能测试|[执行功能测试](../technical-guides/performing-functional-testing.md)|  
|瓶颈测试和优化|[执行测试和优化的瓶颈](../technical-guides/performing-bottleneck-testing-and-tuning.md)|  
|负载测试和最大可持续吞吐量 (MST) 测试|[执行负载测试和吞吐量测试](../technical-guides/performing-load-and-throughput-testing.md)|  
|可用性测试：<br /><br /> -测试单个硬件组件失败。<br />-测试[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]失败。<br />-测试群集节点故障转移。<br />-测试恢复[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]数据库使用 SQL Server 日志传送。|[执行可用性测试](../technical-guides/performing-availability-testing.md)|
