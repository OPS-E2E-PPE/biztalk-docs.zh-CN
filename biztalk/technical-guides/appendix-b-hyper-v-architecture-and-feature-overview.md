 <span data-ttu-id="d9dc0-110">HYPER-V 虚拟机中运行的来宾操作系统提供性能接近在物理硬件上运行的操作系统性能*如果*必要的虚拟服务器客户端 (VSC) 驱动程序和服务来宾操作系统上安装。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-110">Guest operating systems running in a Hyper-V virtual machine provide performance approaching the performance of an operating system running on physical hardware *if* the necessary virtual server client (VSC) drivers and services are installed on the guest operating system.</span></span> <span data-ttu-id="d9dc0-111">HYPER-V 虚拟服务器客户端 (VSC) 代码，也称为 HYPER-V 启用 I/O，支持对 HYPER-V"虚拟机总线"直接访问以及可在安装了 HYPER-V 集成服务。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-111">Hyper-V virtual server client (VSC) code, also known as Hyper-V enlightened I/O, enables direct access to the Hyper-V “Virtual Machine Bus” and is available with the installation of Hyper-V integration services.</span></span> <span data-ttu-id="d9dc0-112">提供 VSC 驱动程序的 HYPER-V 集成服务也已可用于其他客户端操作系统的系统。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-112">Hyper-V Integration services that provide VSC drivers are also available for other client operating systems.</span></span>

 <span data-ttu-id="d9dc0-119">分区的确可以访问物理处理器，也不它们处理处理器中断。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-119">Partitions do not have access to the physical processor, nor do they handle the processor interrupts.</span></span> <span data-ttu-id="d9dc0-120">相反，它们有一个虚拟处理器的视图，专用于每个来宾分区的虚拟内存地址区域中运行。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-120">Instead, they have a virtual view of the processor and run in a virtual memory address region that is private to each guest partition.</span></span> <span data-ttu-id="d9dc0-121">虚拟机监控程序处理对处理器的中断操作，并将他们重定向到相应的分区。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-121">The hypervisor handles the interrupts to the processor, and redirects them to the respective partition.</span></span> <span data-ttu-id="d9dc0-122">此外可以 HYPER-V 硬件加速的地址转换之间通过输入输出内存管理单元 (IOMMU) 进行操作独立于使用的 CPU 内存管理硬件的各种来宾虚拟地址空间。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-122">Hyper-V can also hardware accelerate the address translation between various guest virtual address spaces by using an Input Output Memory Management Unit (IOMMU) which operates independent of the memory management hardware used by the CPU.</span></span> <span data-ttu-id="d9dc0-123">IOMMU 用于重新映射到子分区都使用的地址的物理内存地址。</span><span class="sxs-lookup"><span data-stu-id="d9dc0-123">An IOMMU is used to remap physical memory addresses to the addresses that are used by the child partitions.</span></span>
