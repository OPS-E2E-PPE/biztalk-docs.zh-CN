# <a name="mscit-using-microsoft-azure-biztalk-services-for-supply-chain-orders"></a>MSCIT:使用 Microsoft Azure BizTalk 服务用于供应链订单
**Microsoft 设备 （&) 工作室：使用 Microsoft Azure BizTalk 服务用于供应链订单**  
  
 BizTalk 技术文章  
  
 **编写器：** Anil Kongari (Microsoft)、 Dipti Sharma (Microsoft)、 Mandi Ohlinger (Microsoft)  
  
 **技术审阅人员：** Anil Kongari (Microsoft)、 Hariharan Sundaram (Microsoft)、 Dipti Sharma (Microsoft)、 Heena Parmar (Microsoft)  
  
 **发布日期：** 2013 年 10 月  
  
 **适用于：** Microsoft Azure BizTalk 服务 (MABS) 和 BizTalk Server 2013  
  
 **摘要：** 生产、 供应链和信息服务 (MSCIS) 组是在 Microsoft Global Supply Chain Management 组。 每年，Microsoft 发布新产品。 MSCIS 负责将这些新产品推向市场。 若要支持这些产品，新合作伙伴添加到供应链，包括供应商、 制造商、 分发服务器、 零售商、 服务中心、 运营商，等等。  
  
 每年增加的合作伙伴事务数。 在高负载，有与吞吐量相关的问题。 虽然提供链 BizTalk 中心处理更多的事务 （增加卷），最终系统或合作伙伴系统不无法保持一致。  
  
 当前的挑战是峰值销售准备企业到企业 (B2B) 系统。 作为一部分的技术采用计划 (TAP)，MSCIS 正在研究平台即服务 (PaaS) 使用 Microsoft Azure BizTalk 服务 (MABS)。 MABS 提供可以解决缩放这种情况的重要功能。 MSCIS 创建概念证明 (POC) 涉及使用 Microsoft Azure BizTalk 服务 (MABS) 和 BizTalk Server 2013 的混合解决方案。 要求是提供从端到端，包括对峰值销售扩展和扩展功能下正常销售提供完美的执行。  
  
 本白皮书主要讨论使用 Microsoft Azure BizTalk 服务 (MABS) 和 BizTalk Server 2013 进行测试的解决方案。
