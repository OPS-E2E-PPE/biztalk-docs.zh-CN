# <a name="contents-of-the-biztalk-esb-toolkit"></a>BizTalk ESB 工具包的内容
[!INCLUDE[esbToolkit](../includes/esbtoolkit-md.md)]提供体系结构指南、 模式和实践，以及 BizTalk Server 和.NET Framework 组件，用于简化的开发在 Microsoft 平台上的企业级解决方案的集合。 该工具包还提供功能，可帮助开发人员扩展现有消息传送和集成解决方案。 [!INCLUDE[esbToolkit](../includes/esbtoolkit-md.md)]支持和实现松散耦合的消息传递环境，它简化了生成动态基于消息的企业应用程序的进程的组件进行互操作的集合组成。  
  
 [!INCLUDE[esbToolkit](../includes/esbtoolkit-md.md)]包含以下组件：  
  
-   [ESB Web 服务](../esb-toolkit/esb-web-services.md)。 这些公开到 ESB 使用者，如基于路线的路由、 异常管理、 动态的终结点和地图，BizTalk 操作通用、 描述、 发现和集成 (UDDI) 互操作性，解析的主要功能和消息内容的转换。  
  
-   [ESB 管理门户](../esb-toolkit/esb-management-portal.md)。 这提供对异常和错误跟踪，消息重新提交，警报和通知、 UDDI 集成、 报告和分析，以及配置功能的支持。  
  
-   [ESB 管道互操作组件](../esb-toolkit/esb-pipeline-interop-components.md)。 这些提供 BizTalk Server 管道组件，以方便与外部系统的互操作性。  
  
-   [异常 Management Framework](../esb-toolkit/exception-management-framework.md)。 这将捕获和将合并 BizTalk 消息传送和业务流程的子系统中的异常并提供单一机制，用于处理错误消息。 它包括异常 Web 服务、 异常管理 API 和扩充、 处理和传递给 ESB 管理门户的异常详细信息的组件。  
  
-   [ESB 冲突解决程序和适配器提供程序框架](../esb-toolkit/esb-resolver-and-adapter-provider-framework.md)。 这将实现可插入和可配置体系结构来动态解析终结点和转换要求和实现路由的消息。  
  
-   [路由和处理路线基于](../esb-toolkit/itinerary-based-routing-and-processing.md)。 路线处理机制提供了用于动态描述、 提交，并执行多个服务调用或路由/转换请求一轻型功能。  
  
-   [BizTalk ESB 工具包示例应用程序](../esb-toolkit/biztalk-esb-toolkit-sample-applications.md)。 包含的示例应用程序演示的可能的实现[!INCLUDE[esbToolkit](../includes/esbtoolkit-md.md)]以及您如何使用[!INCLUDE[esbToolkit](../includes/esbtoolkit-md.md)]自己 SOA 和 ESB 的应用程序中的功能。
