# <a name="soa-governance-integration"></a>SOA 管理集成
企业级应用程序必须支持功能强大、 可靠的管理功能，以便能够符合业务要求、 政府立法、 服务级别协议 (Sla) 和客户和贸易合作伙伴的要求。 运行时管理将特别重点，要求和挑战，成功运行面向服务的体系结构 (SOA) – 基于系统满足这些要求。 由业务系统提供的质量是服务的定义其成功或失败的主要因素。  

## <a name="soa-challenges"></a>SOA 挑战  
 基于 SOA 的系统部署到生产环境的企业面临了许多挑战，其中包括：  

-   降低成本的维护和升级，并允许增量更新  

-   允许通过业务流程管理和复合工具快速更改  

-   端到端安全;这包括信任和隐私的消息发送方、 接收方和内容保护  

-   标识、 管理和修复异常发生时  

-   分离的服务和使用者  

-   测量和证明成本关注点的偏移的 SOA 应用程序的业务价值  

-   重复或其它不必要服务的发展壮大的控件 （管理）  

-   促进潜在用户需要降低初始开发成本的相应服务的标识  

-   管理服务，以最大程度减少的成本和风险的不间断的维护和更改的生命周期  

-   简化了适当的服务 （分离的位置、 传输、 策略、 标准和消息传递样式） 的实际使用量  

-   报告功能用于识别谁在使用哪个服务位置和原因
