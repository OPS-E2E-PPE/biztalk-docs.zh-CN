# <a name="viewing-tracked-message-and-instance-data"></a>查看跟踪的消息和实例数据
您可以使用历史记录和跟踪的数据，来帮助您对 BizTalk Server 应用程序中的问题进行疑难解答。 例如，系统管理员可以使用历史记录和跟踪数据完成以下任务：  
  
-   查看在消息流的各个阶段中的跟踪消息事件、消息属性和消息正文。 这些数据可用于排除故障或审核用途。  
  
-   重播特定业务流程实例的执行过程。  
  
-   跟踪规则触发事件以便将来重新构造事件顺序。  
  
-   通过指定标准（如消息架构和消息属性/值对）来查询消息。 使用此方法找到消息后，用户便可确定消息在消息流中已执行到哪一点。 高级用户还可以为消息创建自定义 SQL Server 查询。  
  
-   搜索存档数据库中的存档数据。
