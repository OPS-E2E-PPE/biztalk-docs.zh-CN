# <a name="viewing-tracked-message-and-instance-data"></a>查看跟踪的消息和实例数据
历史记录和跟踪数据可用于帮助解决您的 BizTalk Server 应用程序。 例如，系统管理员可以使用历史记录和跟踪数据：  
  
-   查看在各个阶段中的消息流跟踪的消息事件、 消息属性和消息正文。 此数据可用于排除故障或审核目的。  
  
-   重播特定业务流程实例的执行。  
  
-   跟踪规则触发事件以便将来重新构造在以后的事件的顺序。  
  
-   查询通过指定条件，如消息架构和消息属性/值对的消息。 具有因此位于该消息，用户可确定该消息已发展到的消息流中的点。 高级的用户还可以创建自定义的消息的 SQL Server 查询。  
  
-   搜索存档数据库中的存档数据。
