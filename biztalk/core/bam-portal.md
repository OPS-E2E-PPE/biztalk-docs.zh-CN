# <a name="bam-portal"></a>BAM 门户
业务最终用户使用 BAM 门户监视关键性能指标 (Kpi) 测量进度的业务目标，以及有关其业务流程的其他信息。 业务分析员使用 BAM 门户来监督创建观察模型，是高级别定义的业务流程中的可见性要求。 管理员将其用于各种监视活动，包括监视业务流程管理系统的运行状况。  
  
## <a name="what-is-the-bam-portal"></a>什么是 BAM 门户？  
 BAM 门户提供了向业务流程的实时、 端到端可见性。 它是集合的基于 Web 的功能，它包含的 ASP.NET 页。 您可以自定义 BAM 以提高性能和用户体验。  
  
## <a name="why-use-the-bam-portal"></a>为什么要使用 BAM 门户？  
 BAM 门户允许您以执行搜索、 聚合数据，并对 BAM 视图，这是你的业务数据的角度设置警报。 此可见性可以提供所需的 KPI 信息为您的企业也可以使用概念证明才实现扩展 Microsoft Office Excel 电子表格、 使用 SQL Reporting，或构建一个自定义用户界面 （如另一种解决方案之前UI)。  
  
## <a name="bam-portal-components"></a>BAM 门户组件  
 下面是 BAM 门户组件的简短说明。 有关更详细的说明，请参阅另请参阅中的主题。  
  
### <a name="activity-searches"></a>活动搜索  
 使用 BAM 门户来执行搜索对 BAM 数据进行查找特定 BAM 活动。 通过添加和删除，您可以显示这些活动与你想要收到警报的条件匹配的搜索子句创建查询。  
  
 可保存和重用查询。 它们也可以是一个警报，例如当来自特定客户的采购订单到达时通知的基础。  
  
### <a name="aggregations"></a>Aggregations  
 可通过门户捕获数据的快照并将其显示在图形图表和随附的数据透视表图表的形式。 此数据让你了解该进程或整体业务的运行状况。 例如，用户可能希望看到简单的饼图，显示 1,000 张发票根据处理中的哪个阶段通过每个发票 （400 张仍在"评估"，400 个已拒绝，100 付款，还有 100 仍处于"资金调拨"中） 已达到一天中的细目.  
  
 提供的数据可以是创建一个警报，如"通知我如果过程的评估阶段中有超过 500 的发票。"的基础  
  
### <a name="alert-manager"></a>警报管理器  
 此门户提供有关警报的创建和编辑现有警报用户界面。 警报使用要从活动搜索或聚合页监视的条件。 警报管理器是在警报，如人员通知的相关部分中填充其中如何 （例如，通过电子邮件），以及是否其他人可以查看并订阅警报。
