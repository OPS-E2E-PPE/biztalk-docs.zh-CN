# <a name="before-you-use-the-tpe"></a>使用 TPE 之前
在您开始使用跟踪配置文件编辑器 (TPE) 之前，我们建议您首先了解 BizTalk Server 以及它是如何促进不同平台和应用程序之间的通信的。  
  
 作为解决方案开发人员，您还需要具体了解如何使用 Microsoft BizTalk Server 业务流程来实现您的组织的业务流程。 您必须了解如何使用业务流程设计器来生成业务流程。 有关详细信息[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]业务流程，请参阅[有关业务流程](../core/about-orchestrations.md)。  
  
> [!IMPORTANT]
>  为了使用 TPE，您必须使用完整或自定义的 BizTalk Server 产品安装，并且确保您的选择包括“开发人员工具和 SDK”或“管理工具”选项。 有关详细信息，请参阅[BizTalk Server 2013 和 2013 R2 的安装概述](http://msdn.microsoft.com/library/8041926c-cfc9-4eaf-9c28-a2c6e8015bc5)。  
  
> [!NOTE]
>  TPE 不跟踪 .NET 消息部分。 您不能查看这些类型的消息的消息架构。
