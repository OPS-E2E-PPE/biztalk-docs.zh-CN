# <a name="publishing-schemas-as-a-web-service"></a>将架构发布为 Web 服务
BizTalk Web Services 发布向导用于创建使用现有的架构的 Web 服务。 声明 Web 服务、 Web 方法和你想要发布的请求和响应架构。 使用该向导，你定义的目标命名空间、 SOAP 标头要求和生成的 Web 服务项目的位置。  
  
> [!IMPORTANT]
>  您必须编译包含架构的 BizTalk 程序集。  
> 
> [!IMPORTANT]
>  每个 Web 方法可以具有一个请求架构和一个响应架构。  
> 
> [!NOTE]
>  若要将架构发布为 Web 服务，必须具有[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]和[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]安装。  
> 
> [!IMPORTANT]
>  在运行 BizTalk Web Services 发布向导之前, 必须启用 Web services。 有关启用 Web services 的详细信息，请参阅[启用 Web 服务](../core/enabling-web-services.md)。
