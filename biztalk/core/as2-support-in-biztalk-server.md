## <a name="introduction-to-as2"></a><span data-ttu-id="57933-104">AS2 介绍</span><span class="sxs-lookup"><span data-stu-id="57933-104">Introduction to AS2</span></span>  
 <span data-ttu-id="57933-105">增值网络 (VAN) 是 EDI 的一种常见传输方式。</span><span class="sxs-lookup"><span data-stu-id="57933-105">A common transport used for EDI is Value-Added Networks (VANs).</span></span> <span data-ttu-id="57933-106">增值网络是提供增值服务（例如，精确且合法的绑定审计跟踪）的专有网络。</span><span class="sxs-lookup"><span data-stu-id="57933-106">These are private networks that provide value-added services, such as precise and legally binding audit trails.</span></span> <span data-ttu-id="57933-107">但是，各个公司正在转为通过 Internet 交换 EDI 文档。</span><span class="sxs-lookup"><span data-stu-id="57933-107">However, companies are migrating to exchanging EDI documents over the Internet.</span></span> <span data-ttu-id="57933-108">这样不仅可以降低成本、提高灵活性和效率，而且在冗余性和伸缩性方面具有明显优势。</span><span class="sxs-lookup"><span data-stu-id="57933-108">This lowers costs, increases flexibility and efficiency, and has advantages in terms of redundancy and scalability.</span></span>  
  
 <span data-ttu-id="57933-109">通过 Internet 实现 EDI (EDIINT) 的最常见方法是使用 AS2 (Applicability Statement 2)。</span><span class="sxs-lookup"><span data-stu-id="57933-109">The most common way of implementing EDI over the Internet (EDIINT) is by AS2 (Applicability Statement 2).</span></span> <span data-ttu-id="57933-110">AS2 规范定义了基于 MIME 的安全对等业务数据交换。</span><span class="sxs-lookup"><span data-stu-id="57933-110">The AS2 specification defines MIME-based secure peer-to-peer business data interchange.</span></span> <span data-ttu-id="57933-111">包含带有 MIME 数据的信封的消息使用 TCP/IP 上的 HTTP 进行传输。</span><span class="sxs-lookup"><span data-stu-id="57933-111">Messages containing an envelope with MIME data are transmitted using HTTP over TCP/IP.</span></span>  
  
 <span data-ttu-id="57933-112">AS2 使用 HTTP POST 操作发送 EDI、XML 或其他业务数据。</span><span class="sxs-lookup"><span data-stu-id="57933-112">AS2 uses the HTTP POST operation to send EDI, XML, or other business data.</span></span> <span data-ttu-id="57933-113">AS2 并非只能发送 EDI 数据。</span><span class="sxs-lookup"><span data-stu-id="57933-113">AS2 is not restricted to sending EDI data.</span></span> <span data-ttu-id="57933-114">请求-URI 可标识要用来解压缩并处理消息数据的过程。</span><span class="sxs-lookup"><span data-stu-id="57933-114">Request-URI identifies a process to be used to unpack and handle message data.</span></span> <span data-ttu-id="57933-115">消息处置通知 (MDN) 作为 HTTP 响应消息正文中的确认返回，或者通过发送到原始发送方的 URL 的新的 HTTP POST 操作作为确认返回。</span><span class="sxs-lookup"><span data-stu-id="57933-115">A message disposition notification (MDN) is returned as an acknowledgment either in the HTTP response message body or by a new HTTP POST operation to a URL for the original sender.</span></span>  
  
 <span data-ttu-id="57933-116">有关 EDI 消息传送的详细信息，请参阅[AS2 消息传送](../core/as2-messaging.md)。</span><span class="sxs-lookup"><span data-stu-id="57933-116">For more information about EDI messaging, see [AS2 Messaging](../core/as2-messaging.md).</span></span>
