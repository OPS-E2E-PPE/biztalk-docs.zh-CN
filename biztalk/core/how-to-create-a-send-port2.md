# <a name="how-to-create-a-send-port"></a>如何创建发送端口
本主题介绍如何使用 BizTalk Server 管理控制台创建发送端口。 创建发送端口时，必须选择要创建，请按如下所示的发送端口的类型：  

- 静态单向 — 仅发送端口。  

- 静态要求响应 — 等待目标回复的发送端口。  

- 动态单向 — 仅发送端口可以绑定到的协议和位置在运行时基于消息属性。  

- 动态要求响应 — 等待回复，并且可以绑定到协议和在运行时的位置的发送端口根据消息属性。  

  创建发送端口后，可以执行以下附加步骤以完成配置：  

- 配置高级传输选项，例如，若要重试发送消息，消息失败并对于端口的服务时段计划，如中所述的次数[如何配置传输高级选项的发送端口](../core/how-to-configure-transport-advanced-options-for-a-send-port.md)。  

- 配置备份传输，以防主传输无法正常工作，如中所述[如何为发送端口配置备份传输选项](../core/how-to-configure-backup-transport-options-for-a-send-port.md)。  

- 配置筛选器，以确定消息框中，从哪些消息路由到此发送端口中所述[如何为发送端口配置筛选器](../core/how-to-configure-filters-for-a-send-port.md)。  

- 为要加密或签名由发送端口处理的文档中所述的发送端口分配安全证书[如何为发送端口分配证书](../core/how-to-assign-a-certificate-to-a-send-port.md)。  

- 配置发送端口处理消息的跟踪选项，如中所述[如何为发送端口配置跟踪](../core/how-to-configure-tracking-for-a-send-port.md)。  

## <a name="prerequisites"></a>先决条件  
 若要执行本主题中的过程，必须是 BizTalk Server Administrators 组的成员的帐户登录。 有关详细的权限的信息，请参阅[用于部署和管理 BizTalk 应用程序所需权限](../core/permissions-required-for-deploying-and-managing-a-biztalk-application.md)。 此外，你需要有 SSO 关联管理员在企业单一登录 (SSO) 数据库上的权限。 有关详细信息，请参阅[SSO 用户组](../core/sso-user-groups.md)。  

### <a name="to-create-a-send-port"></a>若要创建的发送端口  

1. 单击**启动**，单击**所有程序**，单击[!INCLUDE[btsBizTalkServerStartMenuItemui](../includes/btsbiztalkserverstartmenuitemui-md.md)]，然后单击**BizTalk Server 管理**。  

2. 在控制台树中，展开 BizTalk 组和 BizTalk 应用程序想要创建的发送端口。  

3. 右键单击**发送端口**，依次指向**新建**，然后单击要创建的端口类型。  

4. 在中**发送端口属性**窗口中，执行以下操作：  


   |       使用此选项       |                                                                                                                                                                                                                                执行的操作                                                                                                                                                                                                                                 |
   |----------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   |       **名称**       |                                                                                                                                                                                            键入新的发送端口的名称。 此名称必须是 BizTalk 组中唯一的。                                                                                                                                                                                             |
   |  **传输类型**  |                                                                                                       从下拉列表中，选择适当的传输类型或传输协议。 如果端口是要求响应端口，仅支持要求-响应的传输类型是出现在列表中。 此属性是仅对静态端口可见。                                                                                                        |
   |    **配置**     |                                                                                          选择传输类型后，单击**配置**以显示**传输属性**对话框，它提供了特定于传输的配置选项。 此属性是仅对静态端口可见。 单击**帮助**配置说明的对话框中。                                                                                           |
   |   **发送处理程序**   |                                                                                                                                                                  从下拉列表中，选择在其运行发送适配器的主机实例。 此属性是仅对静态端口可见。                                                                                                                                                                  |
   |  **发送管道**   |                                            从下拉列表中，选择处理通过此端口发送的消息的管道。 选择管道之后，可以单击旁边的省略号 (**...**) 按钮以显示**配置管道**对话框中，你在其中配置每个实例为此特定端口的管道属性。 单击**帮助**配置说明的对话框中。                                             |
   | **接收管道** | 从下拉列表中，选择处理通过此端口接收的消息的管道。 此外将通过此管道发送到通过此管道接收的消息的响应。 选择管道之后，可以单击旁边的省略号 (**...**) 按钮以显示**配置管道**对话框中，你在其中配置每个实例为此特定端口的管道属性。 此属性是仅对要求响应端口可见。 |


5. 如果要创建要求响应发送端口，在左窗格中，单击**入站映射**并执行以下操作，根据需要重复如果你想要添加多个映射：  


   |      使用此选项       |                                                                             执行的操作                                                                              |
   |---------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   | **源文档** | 从下拉列表中，选择入站映射的源文档。 发送端口可以具有多个映射，但每个映射应具有唯一的源文档。 |
   |       **地图**       |                                     从下拉列表中，选择要将与源和目标文档关联的映射。                                      |
   | **目标文档** |                                              从下拉列表中，选择入站映射的目标文档。                                               |


6. 在左窗格中，单击**出站映射**并执行以下操作，根据需要重复如果你想要添加多个映射：  


   |      使用此选项       |                                         执行的操作                                         |
   |---------------------|--------------------------------------------------------------------------------------------|
   | **源文档** |         从下拉列表中，选择出站映射的源文档。          |
   |       **地图**       | 从下拉列表中，选择要将与源和目标文档关联的映射。 |
   | **目标文档** |         从下拉列表中，选择出站映射的目标文档。          |


7. 完成配置发送端口，单击**确定**。
