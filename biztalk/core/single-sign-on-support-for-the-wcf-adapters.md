# <a name="single-sign-on-support-for-the-wcf-adapters"></a><span data-ttu-id="ef1f4-102">WCF 适配器的单一登录支持</span><span class="sxs-lookup"><span data-stu-id="ef1f4-102">Single Sign-On Support for the WCF Adapters</span></span>
<span data-ttu-id="ef1f4-103">用于 WCF 接收位置或发送端口使用 BizTalk 管理控制台，可以配置企业单一登录 (SSO)。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-103">You can configure Enterprise Single Sign-On (SSO) for use with a WCF receive location or send port by using the BizTalk Administration console.</span></span> <span data-ttu-id="ef1f4-104">本主题介绍 SSO 与 WCF 适配器的工作方式。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-104">This topic describes how SSO works with the WCF adapters.</span></span>  
  
 <span data-ttu-id="ef1f4-105">在企业环境中，其中用户与各种系统和应用程序交互，则很可能在环境不维护通过多个进程、 产品和计算机的用户上下文。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-105">In an enterprise environment, where a user interacts with various systems and applications, it is likely that the environment does not maintain the user context through multiple processes, products, and computers.</span></span> <span data-ttu-id="ef1f4-106">此用户上下文至关重要提供单一登录功能，因为它不需要验证原始请求的发起人。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-106">This user context is crucial to providing single sign-on capabilities, because it is necessary to verify who initiated the original request.</span></span> <span data-ttu-id="ef1f4-107">若要解决此问题的企业单一登录 (SSO) 提供了应用程序可用于获取向发出原始请求的用户相对应的凭据的 SSO 票证 （不 Kerberos 票证）。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-107">To overcome this problem Enterprise Single Sign-On (SSO) provides an SSO ticket (not a Kerberos ticket) that applications can use to get the credentials that correspond to the user who made the original request.</span></span>  
  
 <span data-ttu-id="ef1f4-108">当接收适配器收到一条消息时，适配器可以从 SSO 服务器请求 SSO 票证。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-108">When a receive adapter gets a message, the adapter can request an SSO ticket from an SSO server.</span></span> <span data-ttu-id="ef1f4-109">此加密的票证包含[!INCLUDE[btsWinNoVersion](../includes/btswinnoversion-md.md)]发出请求，并在超时期限的用户的标识。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-109">This encrypted ticket contains the [!INCLUDE[btsWinNoVersion](../includes/btswinnoversion-md.md)] identity of the user that made the request and a time-out period.</span></span> <span data-ttu-id="ef1f4-110">获取该票证后，它是作为属性添加到传入的消息。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-110">After the ticket is acquired, it is added as a property to the incoming message.</span></span> <span data-ttu-id="ef1f4-111">当发送适配器传送消息时，适配器将联系 SSO 服务器发放的 SSO 票证和关联应用程序名称为其适配器尝试检索凭据。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-111">When a send adapter transmits a message, the adapter contacts an SSO server with the issued SSO ticket and an affiliate application name for which the adapter is trying to retrieve a credential.</span></span> <span data-ttu-id="ef1f4-112">SSO 服务器查找目标关联应用程序的用户凭据，然后将凭据返回给发送适配器，用它来将正确验证的消息发送到关联应用程序。</span><span class="sxs-lookup"><span data-stu-id="ef1f4-112">The SSO server looks up the user credential for the target affiliate application, and then returns the credential to the send adapter, which uses it to send an appropriately authenticated message to the affiliate application.</span></span>
