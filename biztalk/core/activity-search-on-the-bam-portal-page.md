# <a name="activity-search-on-the-bam-portal-page"></a>在 BAM 门户网站上的活动搜索页
当用户需要对 BAM 数据，以查找特定进程的特定情况下执行搜索时，业务最终用户、 开发人员和业务分析师可以使用 BAM 门户页。 这些进程被定义为 BAM 活动。 BAM 活动代表在企业中，如采购订单或贷款申请的工作单元。  
  
## <a name="how-an-activity-search-works"></a>活动搜索的工作原理  
 活动搜索，可对 BAM 数据，以查找指定基于跟踪的值和项可在 BAM 视图中，该匹配条件的活动并显示活动，以便可以对其进行编辑，也可以根据其创建警报执行搜索。  
  
 针对你的搜索创建的查询可保存和重用。 您还可以使用它们，如的基础的警报，例如，"通知我指定的客户的采购订单到达的时"。 使用内容页顶部的按钮，来保存、 打开和运行查询，以及设置警报。  
  
 活动搜索，可以找到你想要监视的活动。 一旦创建活动搜索，可以使用该搜索创建警报，它将通知你感兴趣的事件。  
  
## <a name="the-activity-search-page"></a>活动搜索页  
 活动搜索页分为在内容框架内的三个主要部分：  
  
- **查询**:本部分允许用户通过构建搜索子句基于特定的跟踪的数据项来搜索活动。 用户可以添加和删除所需的子句。  
  
- **列选择器**:本部分允许用户指定的项的数据，从那些可在该视图中，若要返回的任何记录如果与搜索条件匹配。  
  
- **结果**:在本部分中显示查询的结果。  
  
  有关活动搜索和活动搜索页的详细信息，请参阅[BAM 门户中的活动搜索](../core/activity-searches-in-the-bam-portal.md)。
