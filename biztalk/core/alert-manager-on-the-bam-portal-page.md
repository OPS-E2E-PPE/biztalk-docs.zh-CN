# <a name="alert-manager-on-the-bam-portal-page"></a>警报管理器在 BAM 门户页
业务用户、 开发人员和业务分析师使用警报管理器在 BAM 门户页上创建或编辑警报。  
  
 警报是由 BAM 基础结构生成的通知，通知收件人存在一组给定的条件。 在门户的警报管理器页提供了用于创建警报和编辑现有警报的用户界面 (UI)。 警报使用的活动搜索页或聚合页从被监视的条件。 警报管理器用于填充中的所有警报，例如要通知的人员、 如何通知他们 （例如，通过电子邮件），以及是否其他人可以查看警报和/或订阅的相关部分。  
  
 订阅警报，可满足警报的条件时收到通知。  
  
## <a name="creating-alerts"></a>创建警报  
 若要创建警报，警报管理器使用访问**设置警报**从活动搜索或聚合页的按钮。 （活动搜索和聚合页是用户在其中定义要监视的条件。 如您可能想要监视单个结果，如已到达从合作伙伴或聚合，例如 500 或更高版本的发票，在计算中的采购订单。  
  
 当你或其他用户需要的事件和工作流详细信息与你的业务关键绩效指标 (Kpi) 相关的通知时创建的警报。 通过填写警报，例如，若要通知，哪些用户的详细信息，如何向他们发出通知 （例如，通过电子邮件），以及其他用户可以看到还是订阅警报创建警报。  
  
## <a name="the-alert-manager-page"></a>警报管理器页  
 BAM 门户的警报管理器页包含三个部分：  
  
-   **警报摘要**:包含一个数据网格，会显示公共警报的视图上下文。  
  
-   **警报详细信息**:显示有关警报的配置的详细信息。 这些详细信息包括警报，包括警报、 所有者和优先级的消息的名称。  在警报详细信息部分用户还可以指定是否可以由其他用户订阅到警报。  
  
-   **订阅**:用户通过使用本部分订阅警报。
