# <a name="who-uses-bam"></a>谁可使用 BAM？
有两种类型的交互的用户具有信息的辅助功能的 BAM： 业务分析人员和业务用户。  
  
 业务分析员使用 BAM Excel 外接程序定义他们需要监视的业务流程。 他们还可以使用 BAM Excel 外接程序定义业务用户查看 BAM 所收集的数据的方式。 例如，业务分析员希望显示有关采购订单的数据，因而提出以下有趣的关于采购订单的问题：  
  
-   今年（本月、本周、今天、这个小时等）公司收到了多少采购订单？  
  
-   所有活动采购订单的当前金额是多少？  
  
-   采购订单从开始到完成采购订单流程的各个阶段平均需要多长时间？  
  
-   流程的某个特定阶段有多少订单？  
  
-   订购特定产品的用户有多少？  
  
-   订购特定产品的订单的总额是多少？  
  
-   各地理区域的订购数量是否有差异？  
  
 用户分析员可以根据上述问题定义活动，反映 BAM 应从业务流程收集哪些类型的信息。  
  
 业务用户使用 BAM 查看业务分析员定义的以及 BAM 收集的数据。 业务用户可在以下方面使用 BAM：  
  
-   查看有关流程的单个实例的数据。 例如，如果业务分析员创建了一个监视采购订单流程的活动，则业务用户可以使用 BAM 来查看有关特定采购订单的数据。  
  
-   查看有关相关活动实例的数据。 例如，与特定采购订单相关的发票或发货。  
  
-   查看有关流程的聚合的实时或存档数据。 聚合的数据是一段时间的数据。 例如，用户可以查看上周（上个月、去年等）收到的、金额少于 $500 的采购订单的数量，或查看今天收到的采购订单的平均金额。  
  
-   查看进度数据，这样用户可以看到流程不同阶段的数据，如采购订单从收到阶段到完成阶段平均经过的时间。
