# <a name="source-control-and-the-biztalk-project-system"></a>源代码管理和 BizTalk 项目系统
源代码管理是一种机制，其中有一个核心软件部分存储和跟踪文件版本，并控制哪些人员可访问这些文件。 使用源代码管理时，你可以执行以下操作：  
  
-   限制重要文件被覆盖的可能性。  
  
-   将版本号应用于文件。  
  
-   存档受源代码管理的文件的旧版本。  
  
-   跟踪文件的修改者、修改时间以及修改内容。  
  
 使用 Visual Studio，你可以与任何 Microsoft 源代码控制接口 (MSSCCI) 兼容的系统相集成。 BizTalk Server 提供支持 Team Foundation Server，并使开发团队能够利用集成的源代码管理、 bug 跟踪、 对于团队开发的支持、 与 Microsoft Office Project Server，以及对自动化支持的集成使用 MSBuild 生成。 BizTalk 项目系统还支持 Microsoft Visual SourceSafe。  
  
 有关源代码管理的详细信息，请参阅在 Visual Studio 结合使用集合中的"源代码管理基础" [http://go.microsoft.com/fwlink/?LinkId=25506](http://go.microsoft.com/fwlink/?LinkId=25506)。
