# <a name="source-control-and-the-biztalk-project-system"></a>源代码管理和 BizTalk 项目系统
源代码管理是一种机制，在其中核心软件部分存储和跟踪文件版本，并控制哪些人可以访问的文件。 当使用源代码管理时，你可以：  
  
- 限制重要文件被覆盖的可能性。  
  
- 将版本号应用于你的文件。  
  
- 存档受源代码管理文件的旧版本。  
  
- 跟踪的文件修改者、 修改时间，以及修改内容。  
  
  Visual Studio 可将与任何 Microsoft 源代码 Control Interface (MSSCCI) 进行集成-兼容的系统。 BizTalk Server 提供了对 Team Foundation Server 的支持并使开发团队能够利用集成的源代码管理、 bug 跟踪、 对团队开发的支持、 与 Microsoft Office Project Server，以及支持自动的集成使用 MSBuild 的生成。 BizTalk 项目系统还支持 Microsoft Visual SourceSafe。  
  
  有关源代码管理的详细信息，请参阅"源代码管理基础知识"Visual Studio 结合使用集合中位于[ http://go.microsoft.com/fwlink/?LinkId=25506 ](http://go.microsoft.com/fwlink/?LinkId=25506)。
