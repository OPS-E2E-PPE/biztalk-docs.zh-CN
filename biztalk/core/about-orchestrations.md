# <a name="about-orchestrations"></a>有关业务流程
业务流程是一个灵活、 强大的工具，用于表示基于 XLANG/s 语言的可执行的业务过程。 XLANG/s 可被视为的表达式功能的某些消息传递语言C#。 可以设计流、 解释和生成数据、 调用自定义代码和组织的整个过程中直观的可视化绘图，并在运行时，BizTalk 业务流程引擎执行 XLANG/s 文件即是可执行业务流程BizTalk 业务流程设计器生成的。  
  
 消息、 发送和接收操作对它们，并通过该传输这些端口是业务流程的所有基本元素。 消息是依据业务流程与外部世界和通信进行电子商务的媒介。  
  
 **接收**并**发送**形状封装到您的业务流程接收消息和从其发送消息所需的功能。 您应熟悉业务流程设计器提供的用于表示您的业务流程的逻辑流的各种形状。  
  
 您应该了解高级业务流程概念，如 Web 服务、 关联和长时间运行的事务。 您可能不需要使用所有这些功能，但最好知道他们可以为您做什么。  
  
 Web 服务是具有遵守规定 Web 服务描述语言 (WSDL) 中的标准的界面的程序。 通过以标准方式定义消息类型、 端口、 端口类型和操作，不同的系统可以彼此有效地通信。  
  
 相关是所依据的消息是与特定运行的业务流程实例相关联的机制，以便您的业务流程获取相应的信息时运行多个实例并且来回发送多条消息。  
  
 事务可以适当地维护业务流程的状态，如果出现意外的问题。 业务流程设计器提供了各种方法来处理异常，以便你能够控制和可预测的方式处理错误。
