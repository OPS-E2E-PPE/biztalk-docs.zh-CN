# <a name="using-policy-explorer"></a>使用策略浏览器
策略浏览器可用于管理策略和规则存储中的规则。 您可以创建、 修改和删除策略和规则，并可以测试、 发布、 部署和取消部署策略。  
  
 下表介绍可以在开发新的策略和规则过程中使用策略浏览器中的命令。  
  
|使用此选项|执行的操作|  
|--------------|----------------|  
|**添加新策略**|创建新的策略 （规则集）。|  
|**添加新版本**|创建所选策略的新空版本。 可以从其他版本复制规则并将其粘贴到新版本。|  
|**复制 （策略版本）**|将所选的策略版本复制到剪贴板。|  
|**复制 （规则）**|将所选的规则复制到剪贴板。|  
|**剪切 （规则）**|将所选的规则复制到剪贴板并将其删除。|  
|**粘贴 （策略版本）**|将策略版本和其内容粘贴到所选的策略。|  
|**粘贴 （规则）**|将规则粘贴到所选的策略版本。|  
|**删除 （策略版本）**|删除所选的策略版本。|  
|**删除 （规则）**|删除所选的规则。|  
|**删除**|删除所选的策略及其所有版本。|  
|**添加新规则**|在所选的策略版本中创建新的规则。|  
|**保存**|保存对所选的版本及其规则所做的任何更改。|  
|**Publish**|发布所选的策略版本。 请注意，不能直接修改已发布的版本。 可以复制并将其粘贴到新版本的策略。|  
|**重新加载**|重新加载所选的策略版本及其规则，可以选择放弃当前对该版本所做的全部更改和从规则存储恢复内容。|  
|**部署**|部署已发布的策略版本。 您必须部署策略版本，以使其可供基于规则的应用程序。|  
|**Undeploy**|取消部署策略版本。 已取消部署的版本后，就不再可用于基于规则的应用程序。|  
|**测试策略**|测试之前部署的所选的策略版本。|  
  
## <a name="properties-window"></a>属性窗口  
 下表介绍用于将策略版本属性。  
  
|属性|ReplTest1|  
|--------------|-----------|  
|**名称**|策略的名称。<br /><br /> 您可以只更改此值，通过更改**名称**策略 （不是策略版本） 的属性。|  
|**事实检索器**|有关策略版本的事实检索器的信息。|  
|**最大执行循环深度**|正向链接算法引发执行循环异常前的最大深度。<br /><br /> 默认的循环次数是 65536。|  
|**转换持续时间**|最长时间之前转换超时异常规则转换会引发。<br /><br /> 默认值为 60000 毫秒。|  
|**在线翻译**|有关转换器用于将转换规则的信息。|  
|**当前版本**|策略浏览器中当前所选策略版本。|  
|**版本说明**|当前版本的说明。|  
  
 下表介绍的规则的属性。  
  
|属性|ReplTest1|  
|--------------|-----------|  
|**在职**|指示是否启用或禁用规则|  
|**名称**|规则的名称。|  
|**Priority**|在策略中规则的优先级。 索引越高，规则优先级越高。 更高的优先级规则的操作将首先触发。<br /><br /> 默认值为 0，表示中等优先级。|
