 <span data-ttu-id="cb515-109">若要使用 SSO，管理员需要定义关联应用程序，其中每个表示非 Windows 系统或应用程序。</span><span class="sxs-lookup"><span data-stu-id="cb515-109">To use SSO, an administrator defines affiliated applications, each of which represents a non-Windows system or application.</span></span> <span data-ttu-id="cb515-110">关联应用程序可能在 IBM 大型机，UNIX 或任何其他类型的软件运行的 SAP ERP 系统运行的客户信息控制系统 (CICS) 应用程序。</span><span class="sxs-lookup"><span data-stu-id="cb515-110">An affiliated application might be a Customer Information Control System (CICS) application that is running on an IBM mainframe, an SAP ERP system that is running on UNIX, or any other kind of software.</span></span> <span data-ttu-id="cb515-111">每个这些应用程序都有其自身的机制进行身份验证，并因此每个都需要其自己唯一的凭据。</span><span class="sxs-lookup"><span data-stu-id="cb515-111">Each of these applications has its own mechanism for authentication, and so each requires its own unique credentials.</span></span>  
  
 <span data-ttu-id="cb515-112">SSO 将存储用户的 Windows 用户 ID 和一个或多个关联应用程序的关联的凭据之间的加密的映射。</span><span class="sxs-lookup"><span data-stu-id="cb515-112">SSO stores an encrypted mapping between the Windows user ID of a user and the associated credentials for one or more affiliated applications.</span></span> <span data-ttu-id="cb515-113">这些链接的对存储在 SSO 数据库中。</span><span class="sxs-lookup"><span data-stu-id="cb515-113">These linked pairs are stored in an SSO database.</span></span> <span data-ttu-id="cb515-114">SSO 以两种方式使用 SSO 数据库。</span><span class="sxs-lookup"><span data-stu-id="cb515-114">SSO uses the SSO database in two ways.</span></span> <span data-ttu-id="cb515-115">第一种方法，称为*Windows 启动单一登录*，使用用户 ID 来确定用户有权访问的关联应用程序。</span><span class="sxs-lookup"><span data-stu-id="cb515-115">The first way, called *Windows-initiated Single Sign-On*, uses the user ID to determine to which affiliated applications the user has access.</span></span> <span data-ttu-id="cb515-116">例如，Windows 用户帐户关联的可能授予对上远程运行的 DB2 数据库访问权限的凭据 / 400 服务器。</span><span class="sxs-lookup"><span data-stu-id="cb515-116">For example, a Windows user account might be linked with credentials that grant access to a DB2 database running on a remote AS/400 server.</span></span> <span data-ttu-id="cb515-117">第二种方法，称为*主机启动的单一登录*，方式相反： 确定哪些远程应用程序有权访问指定的用户 ID，并使用该帐户的权限。</span><span class="sxs-lookup"><span data-stu-id="cb515-117">The second way, called *host-initiated Single Sign-On*, acts in reverse: determining what remote applications have access to a specified user ID, and the privileges that go with that account.</span></span> <span data-ttu-id="cb515-118">例如，远程应用程序关联的可能授予对 Windows 网络具有管理权限的用户帐户访问权限的凭据。</span><span class="sxs-lookup"><span data-stu-id="cb515-118">For example, a remote application might be linked with credentials that grant access to a user account that has administration privileges on a Windows network.</span></span>  
  
 <span data-ttu-id="cb515-119">请注意，SSO 还包含管理工具执行各种操作。</span><span class="sxs-lookup"><span data-stu-id="cb515-119">Note that SSO also includes administration tools to perform various operations.</span></span> <span data-ttu-id="cb515-120">SSO 数据库执行的所有操作进行都审核;例如，工具会提供，使管理员能够监视这些操作并设置不同的审核级别。</span><span class="sxs-lookup"><span data-stu-id="cb515-120">All operations performed on the SSO database are audited; for example, tools are provided that enable an administrator to monitor these operations and set various audit levels.</span></span> <span data-ttu-id="cb515-121">其他工具使管理员能够禁用特定的关联应用程序、 打开和关闭单独的映射是用户，以及执行其他功能。</span><span class="sxs-lookup"><span data-stu-id="cb515-121">Other tools enable an administrator to disable a particular affiliated application, turn on and off an individual mapping for a user, and perform other functions.</span></span> <span data-ttu-id="cb515-122">此外，还有使最终用户能够配置其自己的凭据和映射的客户端程序。</span><span class="sxs-lookup"><span data-stu-id="cb515-122">There is also a client program that enables end users to configure their own credentials and mappings.</span></span>
