# <a name="how-validation-of-an-edi-interchange-is-configured"></a>如何配置 EDI 交换的验证
如果查找协议的过程确定传入或传出交换，解析为协议[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]使用协议属性 （而不是管道属性） 来确定如何执行验证。 如果不存在的协议解析为一个交换，[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]使用某些后备协议属性和一些管道属性中执行验证。  
  
 以下各表指示如何[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]确定用于对 EDI 交换执行的验证。 如果尚未确定协议，则使用管道和回退协议列在表中，则"是"表示该配置。  
  
## <a name="general-information"></a>常规信息  
  
|验证|由贸易合作伙伴配置|按管道配置<sup>*</sup>|配置的协议|由后备协议设置配置|  
|----------------|-----------------------------------|----------------------------------------|-----------------------------|-----------------------------------------------|  
|发送端口关联|是|否|是|否|  
|证书|是|否|否|否|  
|状态报告|否|否|是|是|  
  
> [!NOTE]
>  即使设置都可通过管道，Microsoft 建议在管道上未设置这些属性。
