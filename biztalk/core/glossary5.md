# <a name="glossary"></a>词汇表
Microsoft [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] 的“帮助”中用到以下术语和定义。  
  
## <a name=""></a>。  
  
|术语|定义|  
|----------|----------------|  
|.brl 文件 (.brl file)|业务规则定义文件。|  
|.btm 文件 (.btm file)|BizTalk Server 映射文件。|  
|.btp 文件 (.btp file)|BizTalk Server 管道文件。|  
|.btproj|一种 BizTalk 项目文件。|  
|.btt 文件 (.btt file)|跟踪配置文件。|  
|.odx 文件 (.odx file)|BizTalk Server 业务流程文件。|  
|.xsd 文件 (.xsd file)|BizTalk Server 架构文件。|  
|||  
  
## <a name="a"></a>指向  
  
|术语|定义|  
|----------|----------------|  
|action|与规则的“那么”部分相对应的一个或多个函数，用于指定在条件的计算结果为真时所执行的内容。|  
|激活属性 (Activate property)|接收形状的一种属性，用于指明在收到消息后业务流程被激活。 如果接收形状是业务流程中第一个激活的发送或接收形状，则只能将它标记为激活。|  
|激活块 (activation block)|活动模型内的一组步骤或操作。|  
|激活接收 (activation receive)|接收形状的激活属性设置为 True 时该接收形状的状态的简单描述。|  
|活动模型 (activity model)|预定义的一系列操作。 可以在激活块中调用这些操作。|  
|活动模型步骤 (activity model step)|从活动模型的角度看，活动模型步骤是操作的同义词。 当活动模型内包含操作时，就称为活动模型步骤。|  
|活动节点 (activity node)|指包含里程碑和数据项的节点，其中的里程碑和数据项表示业务分析人员要使用的数据，这些数据由分析人员通过 Microsoft Excel 向导创建。|  
|活动时段 (activity window)|定义在多长时间段内活动实例数据是可见的。 因为时间点始终在变化，所以，该数据也将变化。 在数据经过了该活动时段后，你可以通过运行数据维护 DTS/SSIS 包，将数据移到 BAM 存档数据库。 在你将数据移到 BAM 存档数据库后，这些数据将不再可用于 BAM 门户上的查询。|  
|参与者 (actor)|发起活动或参与活动的人员或进程。 参与者可以是发起人，也可以是目标。|  
|适配器 (adapter)|一种基于 COM 或 .NET 的组件，有助于应用程序（如行业系统）和 BizTalk Server 交换消息。 适配器由执行接收操作和发送操作的设计时组件和运行时组件组成。|  
|适配器框架 (adapter framework)|使用基于 Web Services 的开放标准生成 BizTalk 适配器的规范。|  
|附录 (addendum)|协议的组成部分。 一份协议中包含多个附录，这些附录对以下内容进行定义：使用的业务流程、你的角色、你的合作伙伴的角色、在相应关系中使用的策略或参数，以及诸如业务条款和法律条款列表之类的文档。|  
|关联应用程序 (affiliate application)|企业单一登录 (SSO) 中由管理员定义的一种逻辑实体，表示使用 SSO 连接的系统或子系统（如主机、后端系统或行业应用程序）。 关联应用程序可以表示非 Windows 系统，如大型机或 UNIX 计算机。 它还可以表示应用程序（如 SAP）或系统分支（如“福利待遇”子系统或“付款存根”子系统）。|  
|议程 (agenda)|规则引擎按顺序执行的一系列规则操作。|  
|聚合|包含用于联机分析处理 (OLAP) 多维数据集的预先计算数据的表或结构。 聚合支持针对多维数据库的快速且高效的查询。|  
|协议 (agreement)|关系配置中的定义项。 协议以底层业务流程和参与方管理项目为基础，并根据你自己的配置文件和参与方的配置文件进行定义。 它包含一个或多个附录。 附录定义使用的业务流程、你所在组织的角色、合作伙伴的角色以及关系中使用的参数。 协议还可以指合作伙伴组配置文件，而不是单个的合作伙伴配置文件。 这样便于管理与你所在组织有相同贸易关系的各合作伙伴。|  
|警报通知 (alert notification)|通知用户或用户组可能出现了某种预定义情况的方式。|  
|alias|指代参与方的另一种方式。 每个参与方均被指定了一个默认别名，别名中包含组织名、限定符（如 OrganizationName）和参与方名称的值。 此别名始终被当作默认别名。 别名由名称、限定符和值这三项组成。 同一 BizTalk 管理数据库中任意两个参与方的限定符-值对不能相同。|  
|全部，第一个匹配 (All, FirstMatch)|管道阶段执行模式的两种类型。 “全部”表示以线性模式执行管道阶段中的组件。 “第一个匹配”表示以探测模式执行管道阶段中的组件，能够识别消息格式的第一个组件将使用消息。 在线性模式下，按顺序执行完管道阶段中的所有组件后，才将消息传递到下一阶段。|  
|ANSI X.12|由美国国家标准协会 (American National Standards Institute, ANSI) 制定的一种消息格式。 X.12 这种消息格式主要在美国使用。|  
|应用程序适配器|为与特定应用程序或协议（如 SQL）配合使用而创建的适配器。|  
|存档数据 (archived data)|已由 BizTalk Server 处理并已存储在相应数据库中的数据。|  
|项目 (artifact)|通常表示为 BizTalk 应用程序的一部分，它是 BizTalk 解决方案的逻辑单位。 项目通过 BizTalk 管理控制台管理和部署。 项目的示例包括业务流程、管道、消息架构、安全证书、业务规则策略和绑定等。 项目也是 BizTalk 项目的一部分，它是 BizTalk 解决方案的另一个逻辑单位。 在这一环境中，项目用于在 Visual Studio 中生成构成 BizTalk 解决方案的程序集。|  
|组装器 (assembler)|将单独的文档合成一批文档的管道组件。 BizTalk Server 中提供的组装器管道组件包括平面文件组装器、BizTalk Framework 组装器和 XML 组装器管道组件。|  
|程序集 (assembly)|一个 dll 文件，可能包含要在 BizTalk 应用程序中使用的资源，如业务流程、管道、架构、映射以及其他非特定于 BizTalk Server 的资源。|  
|程序集部署 (assembly deployment)|将程序集资源从 Visual Studio 部署到 BizTalk 应用程序的过程。|  
|程序集部署向导 (Assembly Deployment Wizard)|在 BizTalk Server 2004 中，此向导可引导你完成添加、删除、导入和导出程序集，导入和导出绑定，以及安装程序集或从全局程序集缓存 (GAC) 卸载程序集所需的步骤。 在 [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] 中，“导入向导”、“安装向导”和“导出向导”都不提供此功能。|  
|程序集版本 (assembly version)|程序集的标识符，由主要版本号和辅助版本号组成。|  
|原子业务流程 (atomic orchestration)|一种用于强制提交和回滚 ACID（原子态、一致性、隔离和持续性）事务语义的短期事务，类似于使用 DTC 服务的 COM+ 应用程序。|  
|属性|XML 中用于将附加信息与 XML 元素关联起来的 XML 结构。|  
|受信任验证|标记应用程序的一种方式，指明该应用程序可以使用某一参与方 ID (PID) 将消息提交到 MessageBox 中，该参与方 ID 与应用程序实例服务帐户的 Windows 安全 ID 不同，即参与方 ID 并不是 Windows 安全 ID 的别名。|  
  
## <a name="b"></a>B  
  
|术语|定义|  
|----------|----------------|  
|BAM|请参阅“业务活动监视 (BAM)”。|  
|BAM 活动 (BAM activity)|BAM 为要监视的工作单元捕获的里程碑和数据的端到端列表。 例如，贷款申请活动可能包含里程碑（如“贷款已批准”）和数据（如“申请人姓名”和“贷款金额”）。 BAM 活动通常直接映射到业务流程。|  
|BAM 警报 (BAM alert)|由 BAM 基础结构（特别是 SQL Server Notification Services）生成的通知收件人存在一系列特定条件的通知。 用户定义警报的条件，然后每当满足这些条件时自动服务就生成该警报。|  
|BAM 检查点 (BAM checkpoint)|也称作用于 Excel 的 BAM 加载项中的项。|  
|BAM 配置 (BAM Configuration)|包含 BAM 清单的设置（如 BAM 主导入数据库和服务器名称）的 XML 文件。|  
|BAM 定义 (BAM definition)|BAM 观察模型的 XML 表示形式。|  
|BAM 框架 (BAM Framework)|一组支持动态创建的基础结构和事件集合的托管 API。|  
|BAM 基础结构|由 SQL Server 表、BAM 视图、存储过程和 BAM 数据库（主导入数据库、存档数据库、星型架构数据库和分析数据库）中的数据转换服务 (DTS) 包组成，通过增量部署 BAM 定义来配置和管理。 基础结构是运行时对事件进行关联和聚合并供用户进行查询的地方。|  
|BAM 管理器|管理业务活动监视的动态基础结构的内部组件。|  
|BAM 观察模型 (BAM observational model)|业务流程的可见性要求的高级定义，指定里程碑和要收集的数据事件（BAM 活动）；所有数据聚合的说明；以及将信息呈现给用户（BAM 视图）。|  
|BAM 视图 (BAM view)|从角色特定的角度观察构成 BAM 活动的数据。 视图包括筛选出的数据、所筛选数据的聚合，以及显示所筛选数据的方式（如数据透视图）。 BAM 支持为每个活动定义一个或多个视图。|  
|基本数据项 (base data item)|在 BAM 中，用作维度或度量的基准的活动项。 例如，基本数据项“产品 ID”可以用作“计数”度量值（指定产品的销售量）的基准。|  
|绑定 (binding)|链接软件组件和层的过程。 安装网络组件时，会建立组件间的绑定关系和依存关系。 绑定可使组件之间互相通信。 在 BizTalk Server 中，指的是与业务流程适配器无关的终结点（端口或角色链接）和特定于物理适配器的终结点（发送/接收端口或参与方）之间已建立的映射。|  
|绑定文件 (binding file)|包含绑定时刻快照的文件。 该文件不包括有关业务流程的绑定完整性的详细信息。|  
|BizTalk 管理控制台|用于管理 BizTalk Server 组的 Microsoft 管理控制台 (MMC)。|  
|BizTalk Administrators 组|管理 BizTalk Server 的组。 管理任务包括访问挂起队列、更新配置数据库等。|  
|BizTalk 应用程序 (BizTalk application)|一组为在 BizTalk 管理控制台进行管理而公开的相关项目、资源和设置。 应用程序中的所有项目都可以引用该应用程序中的任何其他项目以及所有引用的应用程序中的任何项目。|  
|BizTalk 应用程序 1|每个新安装的 BizTalk Server 中默认创建的应用程序。 创建此应用程序的目的主要是为了实现向后的兼容性。 已部署但未指定应用程序的项目显示在此文件夹中。 而且，已部署但未指定应用程序的新项目也会部署到该默认应用程序。 通过更改用户设置，可以将任何其他应用程序设置为默认应用程序。|  
|BizTalk Application Users 组 (BizTalk Application Users Group)|一组可以访问特定 BizTalk 组的 MessageBox 的用户。|  
|BizTalk 应用程序视图|打开 BizTalk Server 的系统中心操作管理器控制台后，会显示两个视图之一（连同 BizTalk 部署视图）。 BizTalk 管理员使用该视图监视 BizTalk 项目和应用程序（如业务流程、发送端口和接收位置）的运行状况。|  
|BizTalk 程序集|一个 Microsoft Windows dll 文件，包含 BizTalk Server 业务解决方案中要使用的资源信息，如业务流程、管道、架构和映射。 该程序集还包含版本号、区域性和公钥标记。|  
|BizTalk 部署视图|打开 BizTalk Server 的系统中心操作管理器控制台后，会显示两个视图之一（连同 BizTalk 应用程序视图）。 企业 IT 管理员使用该视图监视 BizTalk Server 安装程序“物理部署”的整体运行状况。|  
|BizTalk 编辑器|一种以 [!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)] 为宿主的可视工具，用于构造可定义 XML 格式和本地格式实例消息的 BizTalk Server 架构。|  
|BizTalk 浏览器|显示 BizTalk 配置数据库内容的 Visual Studio 工具窗口。 该窗口以分层树视图显示程序集、端口和参与方等项。 可以使用 BizTalk 浏览器来配置和管理 BizTalk 项目、参与方和业务流程。|  
|BizTalk 浏览器对象模型|用于创建工具和脚本的 API，以便在 BizTalk 浏览器中自动执行部署后任务。 还可以使用 BizTalk 浏览器对象模型自动执行创建端口、绑定业务流程、管理参与方属性等部署后任务，以及用 BizTalk 浏览器执行的任何其他任务。 BizTalk 浏览器对象模型 API 包含在 Microsoft.BizTalk.ExplorerOM 命名空间中。|  
|BizTalk Framework|一种与平台无关的电子商务框架，它基于可扩展标记语言 (XML) 架构和业界标准。 该框架支持跨行业集成和业务系统间集成，与这些行业和业务系统使用的平台、操作系统或底层技术无关。|  
|BizTalk 组 (BizTalk group)|包含 MessageBox、主机、接收位置、发送端口、发送端口组、业务流程、服务器和适配器的组。|  
|BizTalk 主机 (BizTalk host)|BizTalk Server 中的逻辑进程和安全边界。 每个主机都有一个指定给它的安全组，且可能包含多个主机实例（分别位于不同计算机上）来执行该主机的工作。 而每个主机实例只属于一个主机，且该主机实例的服务帐户属于该主机的安全组。 安全组可用于授予主机中任何主机实例使用物理资源（如数据库）的权限。|  
|BizTalk 管理包警报|一种管理员可以在 BizTalk Server 管理包中订阅的通知。 在订阅一个给定警报后，只要满足某些条件，管理员就将收到通知。 例如，如果一定数量的主机实例正在阻止，则会引发警报。|  
|BizTalk 管理包诊断程序|一种管理员可用来查看给定问题的起因和疑难解答信息的功能。 例如，如果发送端口的运行状况显示为红色，操作管理器控制台中的“状态更改事件”选项卡会显示它从绿色变为红色的原因。|  
|BizTalk 映射程序|一种以 Visual Studio 为宿主的可视工具，用于构造定义数据转换的 BizTalk 映射。|  
|BizTalk 消息队列|BizTalk Server 中的消息队列传输组件。 它是 BizTalk Server 产品的核心内部组件，并且是 BizTalk Server 中的一个适配器组件。|  
|BizTalk 消息库 (BizTalk message store)|用于保存所有消息及其组成部分的 Microsoft SQL Server 表。 使用消息的业务流程可通过此消息库中包含的消息引用，将该消息的副本及其属性从消息库的队列中去除。|  
|BizTalk 项目 (BizTalk project)|Visual Studio 项目的一种类型，用于创建在 BizTalk Server 上运行的应用程序。|  
|BizTalk 项目文件 (BizTalk project file, .btproj)|包含有关 BizTalk 项目的项目特定设置的文件。|  
|BizTalk 项目系统 (BizTalk project system)|用于创建部分或全部的 BizTalk Server 应用程序或业务解决方案的系统。 该系统可以用来添加、编辑或删除 BizTalk Server 项（业务流程、映射、架构和管道）。 它包含编译和部署等命令。|  
|BizTalk Server 管理包|一种 BizTalk 增强功能，有助于确保 BizTalk 应用程序和基础结构的完全监视功能。 它包括诊断和警报这类功能，可有助于监视 BizTalk 部署的运行状况。|  
|BizTalk Server 管理|一个 Microsoft 管理控制台 (MMC) 界面，用于管理由多台服务器组成的 BizTalk Server 组及其属性、监视接收功能并监视服务器组所使用 Microsoft SQL Server 队列中的工作项。|  
|BizTalk Server 映射文件 (BizTalk Server map file, .btm)|BizTalk 映射的持久化形式，该文件由 BizTalk 映射器创建并编译，生成使用可扩展样式表语言转换 (XSLT) 指定的运行时转换指令。|  
|BizTalk 映射程序|一种可使用户在一个 BizTalk 应用程序中的两个架构之间直观地建立转换的功能。 它还具有可用性增强功能，可帮助进行复杂的映射。|  
|BizTalk Server 消息引擎 (BizTalk Server Messaging Engine)|中间件产品中必需的一组服务，用于改进解决方案，使其符合客户环境的要求。 这些运行时服务是 BizTalk Server 平台的基本组成部分。 其中的某些服务是消息的可执行管道处理。 管道处理可将数据格式标准化并具有属性提取功能。|  
|BizTalk Server 管道文件 (BizTalk Server pipeline file, .btp)|描述管道配置以及管道中的组件的文件。 这种文件类型可以编译为 BizTalk 项目的一部分。|  
|BizTalk Server 架构 (BizTalk Server schema)|基于 XML 架构定义语言 (XSD)，对一个或多个 BizTalk Server 实例消息的结构的描述。|  
|BizTalk Server 架构文件 (BizTalk Server schema file, .xsd)|包含 BizTalk 架构的持久化形式的文件。|  
|BizTalk 设置仪表板|一种 BizTalk 管理员可用来对 BizTalk 引擎设置进行集中管理和修改的 BizTalk 功能。 管理员可将设置从一个环境导出和导入到另一环境，如从临时环境到生产环境。|  
|正文部分 (body part)|请参阅“消息正文部分”。|  
|BTSTask|用于管理应用程序和程序集的命令行工具。 可以使用 BTSTask 将 BizTalk 应用程序添加到 BizTalk 管理数据库中，将资源添加到应用程序中，将应用程序导出到 MSI 文件中，将绑定信息导出到文件中，从 MSI 文件导入应用程序，从文件导入绑定信息，列出应用程序中的所有资源，列出 BizTalk 管理数据库中的所有应用程序，列出 MSI 文件的内容，从 BizTalk 管理数据库和 BizTalk 管理控制台删除应用程序，以及从应用程序删除资源。|  
|业务活动监视 (BAM)|BizTalk Server 的一项功能，可为业务用户提供其异构业务流程的实时视图，从而有助于业务用户作出重要业务决策。|  
|业务活动视图 (Business Activity view)|一组显示一个或多个业务流程的分层视图，用于定义某类业务用户希望查看业务活动的方式。 解释一种业务活动数据（BAM 跟踪）的方式可能有多种。|  
|业务分析员 (business analyst)|具备业务管理和经济分析技能的用户。 业务分析员的主要职责是使用业务级数据分析业务趋势。|
