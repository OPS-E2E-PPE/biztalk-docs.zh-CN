 <span data-ttu-id="94380-129">公司 B 创建了自定义的文件移动应用程序，该应用程序可检索文件共享服务器上的平面文件文档，并将这些文档发送到 BizTalk Server 做进一步处理。</span><span class="sxs-lookup"><span data-stu-id="94380-129">Company B has created a custom file-movement application that retrieves the flat file documents from the file share server and sends them to BizTalk Server for additional processing.</span></span> <span data-ttu-id="94380-130">该公司的内部应用程序也可使用此自定义文件移动应用程序，将平面文件传递到 BizTalk Server。</span><span class="sxs-lookup"><span data-stu-id="94380-130">The internal applications for Company B also use the custom file-movement application to pass flat files to BizTalk Server.</span></span> <span data-ttu-id="94380-131">BizTalk Server 将转换这些文档，并将它们发送给该公司的贸易合作伙伴。</span><span class="sxs-lookup"><span data-stu-id="94380-131">BizTalk Server transforms these documents and sends them to Company B's trading partners.</span></span>  
  
 <span data-ttu-id="94380-132">BizTalk Server 将合作伙伴数据转换为内部应用程序格式之前，将验证数据是否具有发件人、收件人以及文档类型条目。</span><span class="sxs-lookup"><span data-stu-id="94380-132">Before BizTalk Server transforms the partner data to the internal application formats, it validates that it has an entry for the sender, receiver, and document type.</span></span> <span data-ttu-id="94380-133">如果 BizTalk Server 接收到没有发件人、收件人或文档类型条目的消息，则会拒绝此消息，并由公司 B 的运作团队对此消息进行审核。</span><span class="sxs-lookup"><span data-stu-id="94380-133">If BizTalk Server receives a message for which it does not have an entry for either the sender, receiver, or document type, BizTalk Server rejects the message, and the operations team of Company B review the message.</span></span> <span data-ttu-id="94380-134">内部应用程序以各种格式发送消息，其中包括 EDIFACT、平面文件、XML 和 ANSI X12。</span><span class="sxs-lookup"><span data-stu-id="94380-134">The internal applications send messages in a variety of formats that include EDIFACT, flat file, XML, and ANSI X12.</span></span>  
  
 <span data-ttu-id="94380-135">公司 B 也通过 HTTPS 从内部源和外部源接收文档。</span><span class="sxs-lookup"><span data-stu-id="94380-135">Company B also receives documents through HTTPS from internal and external sources.</span></span> <span data-ttu-id="94380-136">外部合作伙伴将他们的文档发布在公司网络之外的 Web 服务器上。</span><span class="sxs-lookup"><span data-stu-id="94380-136">External partners post their documents to a Web server outside the corporate network.</span></span> <span data-ttu-id="94380-137">此 Web 服务器与公司网络之间使用防火墙隔开。</span><span class="sxs-lookup"><span data-stu-id="94380-137">A firewall separates this Web server from the corporate network.</span></span> <span data-ttu-id="94380-138">自定义的文件移动应用程序也可通过 HTTPS 检索发布的文档。</span><span class="sxs-lookup"><span data-stu-id="94380-138">The custom file-movement application also retrieves the documents posted through HTTPS.</span></span> <span data-ttu-id="94380-139">公司 B 使用第三方产品对发送给贸易合作伙伴的消息进行加密和签名。</span><span class="sxs-lookup"><span data-stu-id="94380-139">Company B uses a third-party product to encrypt and sign messages to its trading partners.</span></span> <span data-ttu-id="94380-140">作为附加安全措施，公司 B 还对所有服务器进行夜间审核，以确保这些服务器的安全设置正确无误。</span><span class="sxs-lookup"><span data-stu-id="94380-140">As an additional piece of security, Company B performs a nightly audit on all the servers to make sure they have the correct security settings.</span></span> <span data-ttu-id="94380-141">该公司对所有异常情况都进行记录，以备检查。</span><span class="sxs-lookup"><span data-stu-id="94380-141">Company B logs all exceptions for review.</span></span>
