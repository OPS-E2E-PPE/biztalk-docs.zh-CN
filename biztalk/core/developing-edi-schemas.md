# <a name="developing-edi-schemas"></a>开发 EDI 架构
在本部分中的主题介绍如何修改 EDI 架构以用于[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]。  
  
 若要在解决方案中使用文档架构，需要将其添加到的 BizTalk 项目中部署架构[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]，然后生成并部署项目。 [!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)] 将部署默认 BizTalk 应用程序 1 中的项目。 您可以看到通过打开部署的架构**架构**节点下的**BizTalk Application 1**节点**应用程序**中的节点[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]管理控制台。 可以通过使用命令行部署工具部署到不同的应用程序的程序集`BTSTask`。  
  
 会自动部署服务和控制架构[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]配置向导。 若要查看它们，请单击**架构**中的节点**BizTalk EDI 应用程序**在管理控制台中的节点。 有关这些架构的详细信息，请参阅[EDI 服务和控制架构](../core/edi-service-and-control-schemas.md)。
