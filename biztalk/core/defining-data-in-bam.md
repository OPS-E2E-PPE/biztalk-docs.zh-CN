# <a name="defining-data-in-bam"></a>定义 BAM 中的
使用 BAM Excel 外接程序来定义您希望 BAM 收集的数据和定义共享的数据的方式。 使用 BAM 活动定义的数据，并使用 BAM 视图来定义其他用户可以看到的数据。  
  
## <a name="activities"></a>活动  
 创建 BAM 活动定义有关你想要使用 BAM 监视业务流程的信息。 BAM 活动代表业务中的特定业务流程，例如处理采购订单或发运产品。 业务流程具有一组定义的里程碑和业务数据。 例如，采购订单流程可能 Approved、 Denied 和 Delivered 等里程碑以及客户姓名和产品等业务数据。  
  
 BAM 活动的目的是向信息工作者显示历史记录 （里程碑） 和有关进程的数据。 BAM 活动都是独立于 BizTalk Server 的实际实现的高级抽象。 BAM 的概念性概述，请参阅主题"业务活动监视"中[!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]帮助。  
  
 使用 BAM 活动向导可以定义包含至少一个活动项的 BAM 活动。 在活动中的相关的活动项进行分组并使用活动项来描述您想要在业务流程中提供的数据的类型。  
  
 下表描述类型 BAM 提供的活动项。  
  
|项类型|Description|  
|---------------|-----------------|  
|业务里程碑|日期/时间值。 例如，采购订单的批准日期。|  
|业务数据-Text|包含任何字母数字字符的字符串。 例如，发运到：城市、 省/市/自治区和邮政编码的代码。|  
|业务数据 – Integer|一个整数值。 例如，购买的总数。|  
|业务数据-Float|一个十进制值。 有关示例的美元总金额的采购订单。|  
  
 例如，在采购订单活动中，可能会创建以下表中的活动项。  
  
|活动项|项类型|  
|-------------------|---------------|  
|产品|业务数据 — Text|  
|City|业务数据 — Text|  
|State|业务数据 — Text|  
|Amount|业务数据 — Float|  
|Quantity|业务数据 — Integer|  
|已批准|业务里程碑|  
|传递|业务里程碑|  
|拒绝|业务里程碑|  
|收到|业务里程碑|  
  
 请注意量是浮点数，因为它可能是一个十进制值。 数量是一个整数，因为它将始终保持在此示例中的整数。 获得批准，Delivered、 Denied 和 Received 都是采购订单流程中的里程碑。  
  
## <a name="views"></a>Views  
 创建视图以公开对用户活动的数据。 当你创建基于采购订单活动的视图时，您定义活动项背后的数据。 查看数据在 BAM 中定义为维度、 度量值、 持续时间、 里程碑组和进度维度中。  
  
 视图包含一个或多个视图项。 可以创建以下类型的视图项：  
  
-   持续时间  
  
-   里程碑组  
  
-   Aggregations  
  
### <a name="durations"></a>持续时间  
 持续时间为时间间隔。 持续时间的里程碑来定义的开始和结束时间间隔的描述。 下表显示了您可从上表中列出的里程碑的持续时间。  
  
|Duration|开始里程碑|结束里程碑|  
|--------------|---------------------|-------------------|  
|1|收到|已批准|  
|2|收到|传递|  
|3|收到|拒绝|  
|4|已批准|传递|  
  
 在此表中，可以看到第一个持续时间 (持续时间 1) 是启动采购订单由 BizTalk Server 收到并批准采购订单时结束时的时间间隔。  
  
### <a name="milestone-groups"></a>里程碑组  
 您创建里程碑组，例如作为单个实体，将一组里程碑、 开始和结束里程碑的过程中，将创建一个里程碑来表示该过程的整个长度。  
  
### <a name="aggregations"></a>Aggregations  
 使用聚合可以缩短刷新数据库中的数据的响应时间。 Excel 将聚合定义为预先计算的数据汇总，通过在提问之前准备好答案的方式来提高查询响应时间。 例如，在数据仓库事实表中包含数以万计的行时，检索两种特定产品发货计划的查询可能需要很长时间才能完成，因为必须扫描该事实表才能计算出答案。 然而，如果预先计算了答复此查询的汇总数据，则查询几乎可以即时得到响应。  
  
 下图显示的是预先计算的聚合数据的示例。  
  
 ![](../core/media/bam-olap-cube.gif "bam_olap_cube")  
  
 该图汇总了两个月时间段内发送到特定地点的每个产品的数量。 通常，Excel 将此数据定义为度量值。 Excel 将用于筛选和分类的数据定义为维度。  
  
 BAM 工作簿中，可以定义两种类型的聚合：  
  
-   实时聚合  
  
-   计划的聚合  
  
### <a name="real-time-aggregations"></a>实时聚合  
 实时聚合 (RTA) 允许您可以看到业务流程的当前状态，并能够轻松地确定流程瓶颈。  
  
 BAM 数据显示在数据透视表。 您可以将 BAM 透视表定义为 RTA 或计划的聚合。 RTA 可为您提供最新的数据视图，例如，特定 PO 在发货流程中所处的位置。 您可以随时刷新屏幕以更新数据视图。  
  
 在某些情况下，多维聚合的特定切片是时间非常敏感，希望它们在真实时间中不可用。 例如，你的业务销售腐烂的产品和所需的产品数量传递要在真实时间中可用的不同阶段中的聚合函数。 同时，您希望其他聚合，如年龄的典型客户，但只会在商业智能分析的月结束。  
  
> [!IMPORTANT]
>  未定义使用同一个 BAM 活动的多个 Rta。 如果这样做，RTA 数据将不正确，当您存档 BAM 数据。  
  
 有关浏览多维数据的信息，请参阅 Excel 帮助中的数据透视表主题。  
  
### <a name="scheduled-aggregations"></a>计划的聚合  
 所有 BAM 聚合都是默认情况下计划的聚合。 计划聚合表示特定时间的业务快照，例如，今天上午发货活动的摘要。 请询问数据库管理员何时处理聚合，然后您可以查看历史数据。  
  
### <a name="dimensions-and-measures"></a>维度和度量值  
 使用维度和度量值可以创建数据聚合：  
  
- 维度描述事实。  
  
- 度量值是事实的值。  
  
  例如，事实可以是"3 辆红色轿车"清单中。 产品的说明:"car"和"红色"是维度。 事实"3"的值是度量值。 如果每辆轿车的价格包含在这一事实，则轿车价格是维度，但库存中轿车的平均价格是度量值。 Microsoft SQL Server 联机丛书将度量值描述为"的中心值进行聚合和分析。" 换而言之，如果可以对其进行计数、 求平均数，或者执行数学函数以获取它，它是一个度量值。  
  
  可以创建以下类型的维度：  
  
- 进度维度  
  
- 数据维度  
  
- 时间维度  
  
- 数值范围维度  
  
## <a name="progress-dimensions"></a>进度维度  
 BAM 引入了新类型的维度： 进度维度。 创建进度维度可以创建与正在进行的活动的进度相关的聚合。  
  
 例如，考虑你在其中接收 1,000 个采购订单的采购业务流程。 可以在行上使用进度维度来创建下表。  
  
|OrderProgress_Level1|Count|  
|---------------------------|-----------|  
|收到|1000|  
  
 然后可以打开 Received 流程查看更多详细信息有关进度的活动，例如：  
  
|||Count|  
|------|------|-----------|  
|收到|评估|300|  
||已批准|500|  
||拒绝|200|  
  
 这意味着，从你收到了 1000 个采购订单，500 个已批准、 已拒绝 200，300 个当前正在评估。  
  
 Received、 Approved 和 Denied 表示里程碑。 计数列中的相应数字显示多少订单已通过这些里程碑。 评估是订单已接收和 Approved 或 Denied 里程碑之间传递一个阶段。  
  
 与任何其他维度结合使用，可以使用进度维度。 例如，通过在行和数据使用进度维度订单进度维度产品上的列，将出现以下结果：  
  
|||网球拍|足球|  
|------|------|---------------------|------------------|  
|收到|评估|250|50|  
||已批准|200|300|  
||拒绝|150|50|  
  
 进度维度提供的基于实时聚合 (RTA) 的图表非常有用的信息。 Rta 允许您可以看到业务流程的当前状态，并能够轻松地确定流程瓶颈。  
  
 进度维度可以是按顺序进行排序中购买的里程碑： 第一步完成后，才能开始下一步。 或者，里程碑也可以在迁移后。 按顺序的步骤是子步骤，并关联步骤是同级步骤。 在采购订单流程中，验证后将立即开始接收采购订单。 它是接收里程碑时将会发生，因此接收里程碑的同级的短暂步骤。 仅在接收后批准采购订单-已批准的子接收。  
  
## <a name="data-dimension"></a>数据维度  
 定义数据维度，可以在行或列上的 BAM 活动中使用某些文本项的值。 例如，名为产品的数据维度可用于创建下表：  
  
|产品|Count|  
|-------------|-----------|  
|网球拍|100|  
|足球|200|  
  
 此外，您可以在 BAM 视图向导中定义多个数据维度。 例如，定义数据维度州和城市级别命名位置可用来创建以下表：  
  
|产品|Los Angeles|旧金山|Seattle|  
|-------------|-----------------|-------------------|-------------|  
|网球拍|50|20|30|  
|足球|130|50|20|  
  
 在此表中，产品维度用作行和位置维度用作列。  
  
## <a name="time-dimension"></a>时间维度  
 创建时间维度，可以创建有关时间的聚合。 例如，时间维度可以用于创建下表：  
  
|Year|Month|Count|  
|----------|-----------|-----------|  
|2003|January|120|  
||February|230|  
||March|350|  
||April|280|  
  
 可以结合任何其他维度的时间维度。 例如，可以在行和列的数据维度使用时间维度创建以下表：  
  
|Month|网球拍|足球|  
|-----------|---------------------|------------------|  
|January|50|70|  
|February|120|110|  
|March|300|50|  
|April|220|60|  
  
## <a name="numeric-range-dimension"></a>数值范围维度  
 使用数值范围维度可以创建根据友好名称分类的数字范围的聚合。 例如，业务分析员可以定义名 PO 大小为具有以下范围的数值范围维度：  
  
 小，表示采购订单在 0 和 100 美元之间  
  
 中： 表示采购订单在 $100 到 $1,000 之间  
  
 大： 表示采购订单超过 $1,000  
  
> [!NOTE]
>  如果采购订单金额不在定义的范围内，例如，采购订单金额小于 0，则 bam 来容纳超出范围的数据自动创建一个"超出范围"行。  
  
|PO 大小|Count|  
|-------------|-----------|  
|小|500|  
|Medium|350|  
|大型|225|  
  
> [!NOTE]
>  无法创建引用同一数据别名的两个数值范围维度。
