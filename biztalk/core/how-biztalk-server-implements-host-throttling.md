## <a name="components-of-the-host-throttling-algorithm"></a><span data-ttu-id="7b8bd-107">主机阻止算法的组成部分</span><span class="sxs-lookup"><span data-stu-id="7b8bd-107">Components of the host throttling algorithm</span></span>  
 [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]<span data-ttu-id="7b8bd-108"> 在应用主机阻止时使用以下算法：</span><span class="sxs-lookup"><span data-stu-id="7b8bd-108"> uses the following algorithm when applying host throttling:</span></span>

    -   <span data-ttu-id="7b8bd-112">要送达或处理的进程内消息数（用于出站阻止的阈值）。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-112">Number of in-process messages being delivered or processed (threshold for outbound throttling).</span></span>

    -   <span data-ttu-id="7b8bd-114">数据库大小，按所有主机队列表中的项数以及后台处理和跟踪表中的项数来衡量。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-114">Database size, measured by the number of items in the queue tables for all hosts and the number of items in the spool and tracking tables.</span></span>

    -   <span data-ttu-id="7b8bd-122">数据库大小超出阈值。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-122">Database size exceeds threshold.</span></span>  

    -   <span data-ttu-id="7b8bd-123">其他所有阻止条件。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-123">All other throttling conditions.</span></span>  

3.  <span data-ttu-id="7b8bd-124">基于阻止条件的严重性以渐进方式应用阻止。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-124">Progressively apply throttling based upon the severity of the throttling condition(s).</span></span> <span data-ttu-id="7b8bd-125">随着严重级别的提高，也将更为主动地应用阻止。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-125">Throttling is applied more aggressively as the severity level increases.</span></span> <span data-ttu-id="7b8bd-126">渐进式阻止按如下方式来实现：</span><span class="sxs-lookup"><span data-stu-id="7b8bd-126">Progressive throttling is achieved as follows:</span></span>  

    -   <span data-ttu-id="7b8bd-127">检测到一个或多个阻止条件并指定严重性。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-127">One or more throttling conditions are detected and assigned a severity.</span></span>  

    -   <span data-ttu-id="7b8bd-128">基于严重性最高的条件发出实现阻止的指令。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-128">An instruction to implement throttling is issued based on the condition with the highest severity.</span></span> <span data-ttu-id="7b8bd-129">根据阻止条件，如果该条件持续存在，则可能会降低各种线程池的大小，并可能会通过冻结正在运行的业务流程来释放内存。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-129">Depending on the throttling condition, if the condition persists, the size of various thread pools may be reduced and memory may be freed by dehydrating running orchestrations.</span></span>

    -   <span data-ttu-id="7b8bd-139">如果延迟时间已过，则将不再进行阻止。</span><span class="sxs-lookup"><span data-stu-id="7b8bd-139">Throttling is no longer implemented when the delay period has elapsed.</span></span>
