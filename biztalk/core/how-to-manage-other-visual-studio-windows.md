# <a name="how-to-manage-other-visual-studio-windows"></a>如何管理其他 Visual Studio Windows
在 BizTalk 编辑器中的某些操作使用其他 Microsoft[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]窗口，即输出窗口和任务列表窗口。 这些窗口用于显示来自操作，例如生成示例实例消息和验证架构和实例的输出。 本主题提供有关管理这些窗口的分步说明。  
  
### <a name="to-open-the-output-window"></a>若要打开输出窗口  
  
- 在中[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]，单击**视图**菜单，并单击**输出**。  
  
  > [!NOTE]
  >  如果你想要查看输出窗口中的行相对应的文件，请双击该行。  
  
### <a name="to-open-the-task-list-window"></a>若要打开任务列表窗口  
  
- 在中[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]，单击**视图**菜单，并单击**任务列表**。  
  
  > [!NOTE]
  >  如果你想要查看与任务列表窗口中的任务行相对应的 BizTalk 架构中的节点，双击该任务行。  
  
  默认情况下，输出窗口和任务列表窗口共享的同一部分[!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)]shell。 如果在同一时间，这两个窗口处于打开状态，它们将显示包含命名选项卡允许您在它们之间来回切换。
